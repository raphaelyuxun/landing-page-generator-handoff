# 部署与回滚

上线统一走 `scripts/deploy.sh`，回滚走 `scripts/rollback.sh`。核心约定：**每次上线都先把“回滚到上一版所需的一切”快照下来，上线成功后才用新快照替换旧的——任何时刻都恰好保留一份回滚能力。**

## 上线

```bash
scripts/deploy.sh test      # 测试环境 (/opt/easesourcer-test, 4101)
scripts/deploy.sh prod      # 线上     (/opt/easesourcer,      4100)
```

执行流程（任一步失败即中止，已上线步骤可回滚）：

1. 本地构建 `web/dist`
2. **远端快照上一版** → `.rollback-staging/`：`commit.txt`(代码版本) + `web-dist.tgz`(旧前端) + `data.tgz`(数据) + `env.bak`(.env)
3. `git pull --ff-only`
4. `rsync --delete web/dist/`（范围严格限定 `web/dist/`，碰不到 `data/`）
5. 重启 + 健康检查；**失败则自动回滚到上一版，且不替换旧回滚包**
6. 成功后才：`rm -rf .rollback && mv .rollback-staging .rollback`

## 回滚

```bash
scripts/rollback.sh prod                # 默认：只回滚 代码+前端，重启（对业务数据无损）
scripts/rollback.sh prod --with-env     # 额外恢复 .env
scripts/rollback.sh prod --with-data    # ⚠ 额外恢复 data/（丢弃上线后新增任务，仅在数据被损坏时用）
```

回滚**不消耗**回滚包（仍保留，直到下次上线才清理）。

## 数据安全

- `data/`（任务 + 素材）被 `.gitignore` 忽略且 git 不跟踪 → `git pull` 物理上碰不到。
- `rsync --delete` 只作用于 `web/dist/`。
- 重启时在途任务被开机逻辑标记“已中断（可重新生成）”，不损坏已生成的文案/图片/输入；`saveProject` 一次只写单个任务文件，无批量重写。
- 上线前 `data.tgz` 全量快照，必要时可 `--with-data` 完全还原。
