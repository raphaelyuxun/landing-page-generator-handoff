# landing-page-generator 技术交接说明

面向接手研发。本文覆盖架构、部署、代码结构、生成流水线、对接接口、配置、运维与已知限制。配套文档：
[对接接口文档](external-api-spec.md) · [部署与回滚](部署与回滚.md) · [用户使用手册](用户使用手册.md)。

---

## 1. 项目是什么

一个 **B2B Google-Ads 落地页素材生产工作台**：输入（公司名、品类、产品描述、产品图）→ 调 AIGW（文本 + 图像大模型）自动产出落地页素材（Banner + 英文营销文案 + 每个产品的产品图），产物以两份 JSON（`content.json` / `products.json` + 公网图片 URL）交付给外部“落地页渲染平台”（研发侧，`product.easesourcer.com`）发布。

- 两种使用入口：① 运营在**工作台 UI** 手动建/调任务；② 外部平台（云工厂）通过 **/api/ext 接口** 批量建任务、查状态、拉交付。
- **本系统只生产素材，不渲染最终页面**。渲染由研发侧另一个项目负责，双方通过 §7 的 JSON 契约对接。

## 2. 系统架构与部署拓扑

```
                 公网用户/运营                        外部平台(云工厂/研发)
                     │                                      │
      Cloudflare Tunnel (omni-marketeer.com)          HTTPS + X-API-Key
                     │                                      │
   ┌─────────────────┴──────────────────────────────────────┴───────────┐
   │  seo 服务器 (阿里云, Ubuntu 22.04, Node v22)                          │
   │                                                                      │
   │   easesourcer.service        (线上, :4100)   /opt/easesourcer        │
   │   easesourcer-test.service   (测试, :4101)   /opt/easesourcer-test   │
   │        │  Express + tsx 直接跑 TS(无编译步骤)                          │
   │        │  数据落文件系统 data/（无数据库）                              │
   │        ▼                                                             │
   │   AIGW 调用 → 本机 :4500 (relay 模式)                                 │
   └────────────────────┬─────────────────────────────────────────────────┘
                        │ 反向 SSH 隧道 (Mac Mini→seo, 常驻 LaunchDaemon)
                        ▼
   ┌──────────────────────────────────────────────┐
   │  Mac Mini (公司内网)                            │
   │   Python relay :4500 → 注入 AIGW AppKey        │
   │   → 网易 AIGW (文本 claude-opus / 图像 gemini)  │
   └──────────────────────────────────────────────┘
```

**关键点**
- seo 在阿里云外网，**够不到公司内网 AIGW**；Mac Mini 在公司内网、能直连 AIGW。故 seo 把 AIGW 请求发到本机 `:4500`，由 Mac Mini 上的 Python relay 经反向 SSH 隧道接收、**注入 AppKey** 后转发 AIGW。seo 侧代码永远看不到 AppKey，只持有 relay token。
- 也支持 **direct 模式**（部署机本身在内网、直连 AIGW），用 `AIGW_MODE` 切换（见 §8）。
- 入口经 Cloudflare Tunnel：`easesourcer.omni-marketeer.com→:4100`、`easesourcer-test.omni-marketeer.com→:4101`（配置 `~/.cloudflared/config-seo.yml`）。
- 产物图片对外 URL 前缀 = `PUBLIC_BASE_URL`（线上 `https://easesourcer.omni-marketeer.com`），由 `/public/assets/:code/*` 路由提供，公网免鉴权可下载。

**到达 seo 的运维路径**：`ssh seo`（经 bandwagon→macmini ProxyJump）。线上目录 `/opt/easesourcer`，测试 `/opt/easesourcer-test`，两套**完全隔离**（各自 git 检出 / `.env` / `data/` / `web/dist`）。

## 3. 技术栈

| 层 | 技术 |
|---|---|
| 运行时 | Node v22，**tsx 直接跑 TypeScript（无 build/编译产物）**，ESM |
| 后端 | Express；`multer`(上传)、`adm-zip`(解压)、`archiver`(打包导出)、`sharp`(图片压缩/缩放)、`cookie-parser`、`dotenv` |
| 前端 | React + Vite + Tailwind，产物 `web/dist/`（静态，由后端托管） |
| 存储 | **文件系统，无数据库**（`data/projects/*.json` + `data/assets/<code>/`） |
| 鉴权 | 工作台=固定密码 + 签名 Cookie；对接接口=`X-API-Key`（可选 IP 白名单） |
| 进程 | systemd（`Restart=always`），后台任务为进程内异步 + 文件持久化 |

启动命令：`node_modules/.bin/tsx src/server.ts`（systemd `ExecStart`）。本地开发 `npm run dev`（tsx watch）。前端改动需 `cd web && npm run build`。

## 5. 数据模型与存储（无数据库）

一个任务 = 一个 `data/projects/<code>.json`（`GenerationProject`）+ 一个 `data/assets/<code>/` 目录（`raw/` 原图、`images/` 产物图）。核心字段：

- `code`：落地页 URL slug，也是文件名与主键（运营可改名，走 `/rename`）。
- `landingpageId` / `campaignId` / `variantNo`：对接主键与投放任务关联。
- `formInput`：输入（`FormInput`）——公司名、品类、产品描述、`products[]`、`allRawImages`(权威全部输入图)、`imageMeta`(每图名称/描述)、`targetProductCount`(逐任务目标产品数)。
- `state` / `job`：状态机 + 当前后台任务进度（`job.status ∈ running|done|error`）。
- `categoryProfile` / `knobState` / `contentDraft` / `productsDraft` / `assets` / `l2Prompts`：各阶段产物。
- `markedReady` / `delivered` / `archived`：交付/归档标志。
- `apiPayload`：对接接口收到的原始 payload（排查用）。

**并发/一致性**：`saveProject` 每次全量写单个 JSON 文件（无批量重写）。任务进度是**进程内内存态 + 每步 saveProject 落盘**；进程重启会丢失内存中的“running”态 → 开机 `resetStaleJobs` 把残留 running 标为“已中断（可重新生成）”。**单节点、无锁**，靠文件粒度天然隔离。

## 6. 生成流水线（一键全量 = `autorunWork`，可断点续传）

```
下载产品图 → profileCategory(品类画像) → generateCombos(推荐组合)
 → segmentProducts(产品切分) → generateCopy+TrustGuard(文案) → generateImages(banner+g1+产品图)
 → 落盘 assets/URL → state=VALIDATED
```

- **断点续传**：`autorunWork` 若发现 `contentDraft` 已存在则跳过文案链、只补缺失的图；图片复用磁盘已有、只生成缺的。重生成走 `resetForRerun` 清旧产物后全量重跑。
- **产品切分 `segment.ts`（重点）**：把全部输入图（缩到 512px 喂视觉模型）识别成“互不相同的产品”。
  - **产品数无硬上限**；软上限 = 上传图片数（每产品至少 1 图）；逐任务 `targetProductCount` 可再调低。
  - 同一产品多角度 → 合并为 1 个（不凑克隆）。
  - `maxTokens` 随产品上限自适应（`min(16000, 1200+上限×220)`），避免产品多时 JSON 被截断退化。
  - 识别为空时**显式告警**并兜底 1 个产品。
- **图像 `image.ts`**：banner + 参考图 g1 + 每产品 i2i（gemini），带“疑似有文字”QA 检测与一次自动重生。
- **任务看门狗**：`startJob` 内 `Promise.race` 20 分钟兜底，超时判 `error`（可重触发），防止任何未设超时的 await 永久卡住。

## 7. 对外集成接口（研发对接，详见 [external-api-spec.md](external-api-spec.md)）

前缀 `/api/ext/`，鉴权 `X-API-Key`（线上 `esk_live_…`）。三个接口：

| 接口 | 方法 | 说明 |
|---|---|---|
| 创建落地页 | `POST /api/ext/landingpages` | 传 campaign/行业/产品/图片(可带每图 name/desc)，异步生成；`campaign_id` 幂等；可选 `max_products` |
| 批量查状态 | `POST /api/ext/landingpages/status` | 传一组 campaign_id，拿状态/进度/ETA |
| 交付 | `GET /api/ext/landingpages/{id}/delivery` | status=ready 时返回 `content` + `products` 两份 JSON（含公网图片 URL） |

**⚠️ 交付幂等约定（§5.1，重要）**：交付接口每次返回**当前全量**，产品 `id` 稳定为 `{code}-{序号}`。研发入库必须**按 `code` 先删后插 / 或按 `id` upsert**，严禁 append——否则重生成/重复拉取会导致产品累积（曾实测某落地页我方回传 12、研发侧因 append 累积到 56）。

## 8. AIGW 接入（`aigw/client.ts` + `config.ts`）

`AIGW_MODE` 切换：
- `relay`（默认，阿里云外网机用）：请求发到 `AIGW_RELAY_URL`(本机 `:4500`)，带 `X-Relay-Token`；Mac Mini relay 注入 AppKey 转发 AIGW。
- `direct`（部署机在内网时）：直连 `AIGW_BASE_URL`，带 `Authorization: Bearer <AIGW_APP_KEY>`。

`relayPost` 关键实现：`AbortController` 超时**覆盖 fetch + 读 body 全过程**（`clearTimeout` 在 `finally`）——早期版本超时只保护到响应头、隧道传 body 卡住时 `res.text()` 无限挂起，曾导致任务卡死 13 小时，已修。业务错误用 `AigwBusinessError` 标记以区分“不重试”。文本 60s 超时、图像 180s、各 4 次重试。

模型：文本 `TEXT_MODEL`(claude-opus 系)、图像 `IMAGE_MODEL`(gemini-3-pro-image)。

## 9. 配置与环境变量（全在 `config.ts`，真实值只在各机 `.env`，已 gitignore）

| 变量 | 说明 |
|---|---|
| `PORT` | 端口（线上 4100 / 测试 4101） |
| `APP_PASSWORD` | 工作台登录密码 |
| `SESSION_SECRET` | Cookie 签名密钥 |
| `AIGW_MODE` | `relay`(默认) / `direct` |
| `AIGW_RELAY_URL` `AIGW_RELAY_TOKEN` | relay 模式：本机 relay 地址 + token |
| `AIGW_BASE_URL` `AIGW_APP_KEY` | direct 模式：AIGW 地址 + AppKey |
| `TEXT_MODEL` `IMAGE_MODEL` | 模型代号 |
| `DATA_DIR` | 数据根（默认 `./data`） |
| `PUBLIC_BASE_URL` | 产物图片对外 URL 前缀 |
| `SITE_WHATSAPP` `SITE_EMAIL` | 全站固定联系方式（写入每个落地页） |
| `EXT_API_KEYS` | 对接接口接受的 key（逗号分隔） |
| `EXT_IP_ALLOWLIST` | 对接接口 IP 白名单（空=不校验） |
| `MAX_VISION_IMAGES` | 视觉理解喂图数节流阀，**0=喂全部图（默认推荐）**；仅图特别多想省 token 时设正值 |

> 产品数无硬上限配置项（已移除 `MAX_PRODUCTS`）；软上限=图片数，逐任务用“目标产品数”控制。

## 10. 部署与回滚（详见 [部署与回滚.md](部署与回滚.md)）

- 上线：`scripts/deploy.sh <prod|test>` —— 本地构建前端 → 远端**快照上一版**(commit+dist+data+.env → `.rollback-staging`) → `git pull` → rsync dist → 重启 + 健康检查（失败自动回滚）→ 成功后才替换旧回滚包。**任何时刻恰好保留一份回滚能力**。
- 回滚：`scripts/rollback.sh <prod|test> [--with-env|--with-data]`（默认只回代码+前端，对业务数据无损）。
- `data/` 被 gitignore 且 git 不跟踪 → `git pull` 碰不到；`rsync --delete` 只作用于 `web/dist/`。
- 代码托管 GitHub 私有仓库（`landing-page-generator`）；seo 用只读 deploy key 检出。

## 11. 已知限制 / 待办（交接重点）

1. **版式模板硬编码为 m1**：`knobs.ts` 里 `final.template='m1'`（有 TODO），m2 已停用。新增 m3/m4/m5 **需要研发提供每个模板的“板块清单 + 每板块 JSON 字段/新字段 + template id 读取方式 + 素材要求 + 样例 content JSON”**，才能接进生成端。产品侧已定方向：生成时把**所有版式所需内容并集**一次生成、切换版式不重生成。
2. **产品数上限**：无硬上限，软上限=图片数。但**生成 N 张产品图 ≈ N×40 秒**，受 20 分钟看门狗约束，单任务实际能跑完约 ≤25-28 个产品；更多需调大看门狗或用“目标产品数”压低。
3. **交付幂等**：研发侧必须按 `code` 覆盖入库（见 §7），否则产品累积。
4. **单节点 / 无数据库 / 文件存储**：无横向扩展；任务态在内存，进程重启中断在途任务（可 `/autorun` 续跑）。数据备份=打包 `data/`。
5. **文案硬红线**（`trustguard.ts`）：禁编造数值规格/型号/CAS/联系方式；联系方式为全站固定值，不由 AI 生成。

## 12. 常见运维

| 操作 | 命令 |
|---|---|
| 看日志 | `ssh seo "journalctl -u easesourcer -n 100 -f"` |
| 重启 | `ssh seo "sudo systemctl restart easesourcer"` |
| 查某任务 | 读 `/opt/easesourcer/data/projects/<code>.json` 的 `state/job/logs` |
| 重跑卡住的任务 | 登录拿 Cookie 后 `POST /api/projects/<code>/autorun`（15 分钟以上的 running 会被接管） |
| relay 是否通 | `ssh seo "curl -s :4500/healthz"`（200=通） |
| 备份数据 | `ssh seo "tar czf ~/data-$(date +%F).tgz -C /opt/easesourcer data"` |

## 13. 密钥管理

所有真实密钥**只存在于各机 `.env`（已 gitignore）**，源码 / `.env.example` / 文档一律占位符。AIGW AppKey 只在 Mac Mini relay 侧持有，seo 代码看不到。对接 `esk_live_…` key 妥善保管；如泄露可轮换（改各机 `.env` 的 `EXT_API_KEYS` + 通知研发换新）。
